# Implementation Plan / 实施计划

This plan is adapted from the original classroom Skill proposal and refined for a GitHub-ready documentation project.

## Phase 1: Foundation

| Step | Task | Output |
|---|---|---|
| 1 | Learn basic theories: cross-cultural communication, semiotics, localization, cultural dimensions | `docs/theory-notes.md` |
| 2 | Define target use cases | `docs/use-cases.md` |
| 3 | Define risk dimensions and rating rules | `data/starter-risk-taxonomy.md` |

## Phase 2: Skill Design

| Step | Task | Output |
|---|---|---|
| 4 | Write the core Skill instructions | `SKILL.md` |
| 5 | Create input and output templates | `templates/` |
| 6 | Create sample reports | `examples/` |

## Phase 3: Testing

| Step | Task | Output |
|---|---|---|
| 7 | Test with known public ad controversy cases after verifying sources | `examples/case-tests/` |
| 8 | Test with low-risk fictional ads | `examples/low-risk-tests/` |
| 9 | Revise scoring, warnings, and output clarity | `CHANGELOG.md` |

## Phase 4: Publishing

| Step | Task | Output |
|---|---|---|
| 10 | Add README, disclaimer, and license | `README.md`, `LICENSE` |
| 11 | Publish to GitHub | Public repository |
| 12 | Collect issues and improve | GitHub Issues / Pull Requests |

## Quality Checklist

- [ ] Does the skill avoid cultural stereotyping?
- [ ] Does it distinguish confirmed facts from possible risks?
- [ ] Does it provide safer alternatives instead of only warnings?
- [ ] Does it include local validation steps?
- [ ] Does it avoid legal advice?
- [ ] Can a student or junior marketer use it without extra explanation?
